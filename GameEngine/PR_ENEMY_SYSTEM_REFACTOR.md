# [PR] 적(Enemy) 시스템 최적화 및 상태 시스템 간소화 보고 (2026-05-23)

## 📝 개요
본 작업은 적(Enemy) 시스템의 비효율적인 구조를 전면 개편하고, 팀장님의 **State System 리팩토링 가이드**를 준수하여 유지보수성을 극대화한 작업입니다. 특히 불필요한 상태를 제거하는 '상태 다이어트'를 통해 시스템을 경량화했습니다.

---

## 🚀 주요 추가 및 변경 사항

### 1. 오브젝트 풀링(Object Pooling) & 크래시 방지
*   **리소스 선할당**: 게임 시작 시 30마리의 적을 미리 생성하여 런타임 중 GPU 버퍼 생성 및 메모리 할당 부하를 제거했습니다.
*   **메모리 안전성**: 업데이트 루프 중 리스트 수정으로 인한 반복자 무효화(Iterator Invalidation) 크래시를 방지하기 위해, 정적 풀 구조를 도입했습니다.

### 2. 상태 시스템 간소화 (상태 다이어트)
*   **3대 핵심 상태 확립**: 기존의 복잡한 상태들을 제거하고 적 전용 핵심 상태 3종(`Move`, `Dead`, `Disabled`)으로 재편했습니다.
    *   `Idle`, `Attack`, `Hit` 상태 제거: 현재 AI 로직(플레이어 상시 추적)에 불필요한 상태를 과감히 삭제하여 로직을 단순화했습니다.
*   **소환 즉시 활성화**: 풀에서 나오는 즉시 `Move` 상태로 전환되어 지연 없이 플레이어를 추격합니다.

### 3. 리팩토링 가이드 준수 및 중앙 집중화
*   **콜백 통합**: `EnemyCallbacks`를 삭제하고 모든 반응 로직을 `Callbacks/StateCallbacks.cpp`로 통합하여 가이드의 "중앙 집중 관리" 원칙을 완벽히 준수했습니다.
*   **주석 명시**: 통합된 코드 섹션에 작업 날짜(2026-05-23)와 사유를 명시하여 팀원 간의 소통 효율을 높였습니다.

## 🛠 트러블슈팅 (Troubleshooting)

### 이슈 1: 초기 위치 유령(Ghost) 현상 해결
*   **원인**: 생성 직후 `CollisionSystem`이 화면 밖 오브젝트를 경계 안으로 강제 스냅(Snap)하여 우측 상단에 유령 이미지가 남는 현상.
*   **해결**: 시스템이 감시하지 않는 **Z축 변경(Z=10.0f)** 기법을 적용하여 비활성 객체를 완벽히 숨겼습니다.

### 이슈 2: 3D 거리 계산으로 인한 충돌 감지 불가 해결
*   **원인**: 은닉용 Z축 값이 스폰 후에도 남아있어 플레이어와 슬라임 간의 3차원 거리가 멀게 측정됨.
*   **해결**: 스폰 시 Z축을 플레이어와 동일한 `0.0f`로 정확히 리셋하여 충돌 판정을 정상화했습니다.

### 이슈 3: 범용 충돌 알림 로직 부재 해결
*   **원인**: 기존 시스템이 "Bullet" 이름만 체크하여 슬라임 충돌 시 `lose!` 메시지가 출력되지 않음.
*   **해결**: `CollisionSystem`에 "Enemy" 네임태그 체크 로직을 추가하여 모든 적 유닛에 대응하도록 확장했습니다.
